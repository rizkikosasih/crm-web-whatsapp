‚úÖ LANGKAH 1: Masuk ke Google Cloud Console
Buka: https://console.cloud.google.com/
Pastikan kamu sudah login dan memilih project yang aktif (misalnya: crm-wa-460906)

‚úÖ LANGKAH 2: Aktifkan Google Drive API
Jika belum aktif:

1. Buka: https://console.cloud.google.com/apis/library
2. Cari: Google Drive API
3. Klik: Enable

‚úÖ LANGKAH 3: Buat Service Account
1. Buka menu: IAM & Admin > Service Accounts
   Atau langsung ke: https://console.cloud.google.com/iam-admin/serviceaccounts
2. Klik: Create Service Account
3. Isi:
   - Service account name: misalnya crm-drive-uploader
   - Service account ID akan otomatis terisi
4. Klik: Create and Continue
5. (Optional) Tambahkan role: Editor atau Storage Admin
6. Klik: Done

‚úÖ LANGKAH 4: Buat Kunci (Key) JSON
1. Cari service account yang sudah dibuat (misalnya crm-drive-uploader@...)
2. Klik ikon tiga titik di kolom kanan ‚Üí Manage keys
3. Klik Add Key ‚Üí Create new key
4. Pilih: JSON ‚Üí Klik Create

‚û°Ô∏è Simpan file JSON ke dalam proyek Laravel Anda di:
   storage/app/public/google-service-account.json

Jika ingin pakai terminal (Linux/macOS):

mv ~/Downloads/your-service-account-file.json storage/app/public/google-service-account.json

(Ganti nama file sesuai dengan yang Anda unduh)

üß© Agar Service Account Bisa Akses Folder Google Drive Pribadi:

Service account TIDAK otomatis bisa akses folder Google Drive user.
Solusi:

1. Buat folder di Google Drive Anda
2. Klik kanan ‚Üí Share
3. Masukkan alamat client_email dari service account
   Contoh: crm-drive-uploader@project-id.iam.gserviceaccount.com
4. Beri akses: Editor

‚úÖ Sekarang service account bisa upload ke folder tersebut.

‚û§ MENDAPATKAN FOLDER ID dari URL Google Drive

1. Buka folder Google Drive di browser
2. Contoh URL:
   https://drive.google.com/drive/folders/1A2B3C4D5E6F7G8H9I
3. Maka folder ID adalah:
   1A2B3C4D5E6F7G8H9I

üìå Tambahkan ke file .env Laravel Anda:
GOOGLE_DRIVE_FOLDER_ID=1A2B3C4D5E6F7G8H9I
